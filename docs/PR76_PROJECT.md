## Phase 1: Code Changes

### 1.1 Add fsync before rename in `persist_binlog_state()` [High]

**File:** `src/component_src/myvector_binlog_service.cc`  
**Requirement:** RFC-004 specifies "write temp → fsync → rename" for atomic persistence.

**Current:** Uses `std::ofstream` + `std::rename()`; no fsync before rename.

**Steps:**
1. After `out.close()` and before `std::rename()`, open the temp file with `open()` and call `fsync(fd)`.
2. Close the fd before rename.
3. Handle cross-platform: use `#ifdef _WIN32` for `_commit()` (see `src/myvector.cc:57`) or `fsync()` on POSIX.

**Reference:** `docs/PR76_REVIEW_RESPONSE.md` §2, `docs/rfc/RFC-004-issue-body.md`

---

### 1.2 Implement charset metadata for component UDFs [Medium]

**File:** `src/component_src/myvector_udf_service.cc`  
**Requirement:** Match plugin behavior: set `result_set(initid, "charset", latin1)` for UDFs that return string/binary.

**Current:** Line 119 has the call commented out. `myvector_component_udf_metadata` is set in `myvector_component.cc` and used via `MYVECTOR_UDF_METADATA()` in `src/myvector.cc` for plugin; component UDF service uses its own init path.

**Steps:**
1. In `myvector_ann_set_init` (and any other UDF init that returns string/binary), call `myvector_component_udf_metadata->result_set(initid, "charset", latin1)` when the service is available.
2. Ensure `myvector_component_udf_metadata` is non-null before calling (it is set during component init).
3. Verify all UDFs that need charset metadata: compare with `src/myvector.cc` (lines 1613, 1935).

**Reference:** `docs/PR76_REVIEW_VERIFICATION.md` §6
