# Design: Adopt MySQL 9.7 LTS for Component Build

**Date:** 2026-04-23
**Branch:** feature/myvector-component
**Status:** Approved

---

## Problem

The `build-component-9-6` CI job has been failing through multiple layers:

1. `libmysqlclient.so` not present in `mysql:9.6` Docker image — attempted external RPM install, which used an unreliable URL
2. RPM URL failing → switched to building `libmysqlclient` from MySQL source (`make mysqlclient`)
3. `make mysqlclient` produces `.a`, not `.so` — find command searched wrong extension
4. After fixing find: `fatal error: mysql/mysql.h: No such file or directory` — MySQL 9.6 source tree no longer places the C client header at `include/mysql/mysql.h`

In parallel, MySQL 9.7.0 LTS was released with direct CDN downloads for el9 and an official `mysql:9.7` Docker image. Adopting 9.7 LTS supersedes the 9.6 fixes entirely.

---

## Goal

Replace the broken `build-component-9-6` CI job with a clean `build-component-9-7` job that:
- Installs `mysql-community-devel` from MySQL's CDN (version-pinned, no repo setup)
- Builds the MyVector component against MySQL 9.7 LTS headers and library
- Tests it against the official `mysql:9.7` Docker image

---

## Design

### 1. New build script: `scripts/build-component-9.7-docker.sh`

Replaces `scripts/build-component-9.6-docker.sh` entirely. Runs inside `oraclelinux:9` container.

**Step 1 — Install MySQL 9.7 devel RPMs from CDN**

Four RPMs installed in dependency order, directly from `cdn.mysql.com`. Architecture-aware via `$(uname -m)` — covers both x86_64 (current CI runners) and aarch64:

```
https://cdn.mysql.com//Downloads/MySQL-9.7/mysql-community-common-9.7.0-1.el9.${ARCH}.rpm
https://cdn.mysql.com//Downloads/MySQL-9.7/mysql-community-client-plugins-9.7.0-1.el9.${ARCH}.rpm
https://cdn.mysql.com//Downloads/MySQL-9.7/mysql-community-libs-9.7.0-1.el9.${ARCH}.rpm
https://cdn.mysql.com//Downloads/MySQL-9.7/mysql-community-devel-9.7.0-1.el9.${ARCH}.rpm
```

This installs:
- Headers to `/usr/include/mysql/` (including `mysql.h`, `plugin.h`, component headers)
- Library to `/usr/lib64/mysql/libmysqlclient.so` (and `.a`)

No Yum repo RPM needed — direct CDN URLs are version-pinned and confirmed accessible.

**Step 2 — Clone MySQL 9.7 source**

Still required for:
- Component framework headers (`mysql/components/component_implementation.h`, etc.) that may differ between source and installed package
- Binlog event headers (`mysql/binlog/event/binlog_event.h`)
- cmake configure step that generates derived headers (`mysqld_error.h`, `mysql_version.h`, etc.)

Same skip-if-cached logic as the 9.6 script (checks for `.git` and a sentinel header file).

**Step 3 — cmake configure (header generation only)**

Same flags as the 9.6 script: gcc-toolset-14 compilers, DOWNLOAD_BOOST, all optional components disabled. No `make` invocation after configure — the library comes from the RPM.

**Step 4 — Build myvector component**

Locate `libmysqlclient` from the installed RPM:
```bash
MYSQLCLIENT_LIB=$(find /usr/lib64 /usr/lib \( -name "libmysqlclient.so" -o -name "libmysqlclient.a" \) 2>/dev/null | head -1)
MYSQL_LIBDIR=$(dirname "$MYSQLCLIENT_LIB")
```

Pass to cmake:
```cmake
-DMYSQL_SOURCE_DIR="$MYSQL_SRC"
-DMYSQL_BUILD_DIR="$MYSQL_SRC/bld"
-DMYSQL_DIR="$MYSQL_LIBDIR"
```

**Header resolution for `mysql/mysql.h`:**

The explicit cmake include path `-I${MYSQL_SOURCE_DIR}/include` is searched first. MySQL 9.7 source does NOT have `include/mysql/mysql.h`. GCC then falls through to the system include path, finding `/usr/include/mysql/mysql.h` from the installed devel RPM. No symlinks or stubs needed.

### 2. `CMakeLists.txt` — add `lib64/mysql` to PATH_SUFFIXES

MySQL RPMs on el9 install the library to `/usr/lib64/mysql/`. The current `PATH_SUFFIXES` list (`lib lib64 lib/mysql`) doesn't cover this path. Add `lib64/mysql`:

```cmake
find_library(MYSQLCLIENT_LIBRARY
  NAMES mysqlclient
  HINTS "${MYSQL_DIR}" "${MYSQL_BUILD_DIR}" "${MYSQL_SOURCE_DIR}"
  PATH_SUFFIXES lib lib64 lib/mysql lib64/mysql
)
```

This also benefits the 8.4 component build and any future builds that use an el9 RPM-installed library.

### 3. `.github/workflows/ci.yml` — update 9.6 → 9.7

| Item | Old | New |
|---|---|---|
| Build job name | `build-component-9-6` | `build-component-9-7` |
| Test job name | `test-component-9-6` | `test-component-9-7` |
| Script | `build-component-9.6-docker.sh mysql-9.6.0` | `build-component-9.7-docker.sh mysql-9.7.0` |
| Source cache key | `mysql-source-9.6.0-v1` | `mysql-source-9.7.0-v1` |
| Source cache path | `mysql-server-9.6` | `mysql-server-9.7` |
| Artifact name | `myvector-component-mysql9.6-linux` | `myvector-component-mysql9.7-linux` |
| Test Docker image | `mysql:9.6` | `mysql:9.7` |
| Test `needs:` | `build-component-9-6` | `build-component-9-7` |

`continue-on-error: true` remains on the test job.
