# Design: Fix MySQL 9.6 Component Build — gcc-toolset-14

**Date:** 2026-04-22
**Branch:** feature/myvector-component
**Status:** Approved

---

## Problem

`build-component-9-6` CI job fails at the MySQL source cmake step with:

```
CMake Error: Could not find devtoolset compiler/linker in /opt/rh/gcc-toolset-14
Or you can set CMAKE_C_COMPILER and CMAKE_CXX_COMPILER explicitly.
```

MySQL 9.6's cmake detects OracleLinux 9 and hard-requires `gcc-toolset-14`. The script
installs base `gcc`/`g++` (GCC 11.5) but not the toolset. cmake aborts before generating
headers.

---

## Goal

Make `build-component-9-6` pass reliably by installing gcc-toolset-14 and pointing cmake
at it for both the MySQL header-generation step and the myvector component build step.

---

## Design

### 1. Add gcc-toolset-14 packages to `dnf install`

The `ol9_codeready_builder` repo is already enabled. Add to the existing install block:

```bash
gcc-toolset-14-gcc gcc-toolset-14-gcc-c++ \
gcc-toolset-14-binutils \
gcc-toolset-14-annobin-annocheck gcc-toolset-14-annobin-plugin-gcc
```

Base `gcc`/`g++` stay — they are used implicitly by `dnf` and other tooling.

### 2. Pass `-DCMAKE_C_COMPILER` / `-DCMAKE_CXX_COMPILER` to both cmake invocations

**MySQL source cmake** (generates headers in `$MYSQL_SRC/bld`):

```cmake
-DCMAKE_C_COMPILER=/opt/rh/gcc-toolset-14/root/usr/bin/gcc
-DCMAKE_CXX_COMPILER=/opt/rh/gcc-toolset-14/root/usr/bin/g++
```

**myvector component cmake** (`cmake -B build -S .`):

Same two flags — ensures the `.so` is compiled with the same toolchain as the headers.

### 3. What does NOT change

- `ol9_codeready_builder` enablement (already in place)
- `oraclelinux:9` base image (correct for ABI compatibility with mysql:9.6.0)
- `libmysqlclient` extraction from `mysql:9.6.0` image
- All other cmake flags
- CI job structure and `continue-on-error: true`
