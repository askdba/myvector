## Why We Are Still at This Stage

### 1. CI Failures Block Merge

Two distinct CI failure modes:

**A. build-component-9-6**  
- Script: `scripts/build-component-9.6-docker.sh`  
- Uses `oraclelinux:9` + libs extracted from `mysql:9.6.0`  
- Fails during `dnf install -y ...` (exit code 1)  
- Likely cause: dnf/mirror timeout, missing package, or oraclelinux:9 environment change  
- Impact: MySQL 9.6 component build not validated in CI

**B. test (plugin)**  
- All three plugin test jobs (8.0, 8.4, 9.0) fail in the **reinstall** step  
- Sequence: DROP FUNCTIONs → UNINSTALL PLUGIN → INSTALL PLUGIN → CREATE FUNCTIONs  
- Error: `Function 'myvector' already exists` (occurs on INSTALL PLUGIN or first CREATE FUNCTION)  
- CI only drops 4 UDFs; `sql/myvectorplugin.sql` registers 7 (including `myvector_search_open_udf`, `myvector_row_distance`, `myvector_ann_set`)  
- Possible causes:  
  - Plugin or MySQL registers a function named `myvector` that is not dropped  
  - Incomplete DROP list in CI (missing UDFs)  
  - MySQL version-specific behavior on UNINSTALL PLUGIN / UDF cleanup
